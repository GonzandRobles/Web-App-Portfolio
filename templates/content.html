Client Side Validation.- This is where instant feedback is generated on a field-to-field basis without submission. Javascript it's used here. This one offers a more friendly user support because it tells you right away if you put something good or wrong and you don't have to submit the form to know what is good or wrong. The con of this validation is that if the user has the javascript off, they will not see the validation message.

As a conclusion, the best validations are the ones that anticipate errors that user might make. That's why it is a good idea not to leave the validation forms at the end because once the code is complete, it might be very difficult to adapt it.

JSON

JSON means JavaScript Object Notation, the reason why it has JavaScript in it's name is because it is a valid JS code; its a simple format that allows you to store and transfer nested or hierarchal data; it's so popular that other languages has libraries capable to parse and write JSON like Python JSON library. It's easy to bring bug inside JSON files when written by hand that's why it's recommendable to use JSON Lint to verify sintax errors.
JSON serves the same purpose as XML and that is to exchange codes in readable way between humans and computers in a constant format. It can contain dictionaries, objects, lists, booleans, null, floats, numbers, strings...etc.
Let's see an example of who JSON looks:
1- >>> import json
2- >>> j = '{"one": 1, "numbers": [1,2,3.5]}’
3- >>> json.loads(j)
4- {u'numbers': [1, 2, 3.5], u'one': 1}
5- >>> d = json.loads(j)
6- >>> d['numbers’]
7- [1, 2, 3.5]
8- >>> d['one’]
9- 1
If we breakdown the code, then in the 1st line we import JSON
then in the second line we have a variable named j that has inside a string that contains a dictionary with two keys "one" and "numbers"; the value of one is an integer, the value of numbers is a list with integers and a float
in the 3rd line we call a function named load string "loads"
4th line loads a python dictionary with our keys and values, Python dictionaries doesn't return values in the same order that JSON has them.
5th lines saved json.loads(j) in a new variable named d
6th line calls d with the value of the dictionary numbers
the line 7 gets us the dictionary inside key "numbers"
we call d with key "one" in line 8
the last line, 9th shows us the value of key "one", 1

On the first part we have a TextView that is conformed by text, color, background, width and height attributes; if we look closely to the attribute textColor we can see that it's value is an hex color, the background has an hex color too, all the values are inside quotes just like strings. We also find booleans here, take a look to android:textAllCaps="false", the value is false but if we switch that value for true, then all the text is going to be uppercase. In the ImageView case we have an interesting attribute named android:scaleType="centerCrop" and this attribute it's going to adjust the center of the image and fit it into the screen so you can see the complete image instead of just a portion of it; it also gives this look of no border image, that is called full bleed image. We also learn that there are some default font sizes in android dev, you only need to apply the codes below to switch between small, medium and large.

android:textAppearance="?android:textAppearanceSmall"
android:textAppearance=“?android:textAppearanceMedium"
android:textAppearance="?android:textAppearanceLarge"
